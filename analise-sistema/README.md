## 📝 Descrição da Atividade
Esta atividade tem como objetivo aplicar conceitos fundamentais de **Engenharia Reversa**, **Arquitetura de Software** e **Modelagem UML** através da análise de um sistema em produção real.

O objeto de estudo escolhido é o sistema de pedidos da **Tropykaly Pizzas e Lanches**. Diferente de sistemas didáticos, este projeto foi analisado sob a perspectiva de comportamento (black-box), uma vez que o código-fonte original não está disponível.

### Objetivos Principais:
* **Análise Comportamental:** Identificar o objetivo do sistema, suas funcionalidades e fluxos de interação do usuário.
* **Engenharia Reversa:** Inferir a arquitetura e o design de software a partir da interface e funcionamento do site.
* **Modelagem UML:** Identificar entidades, definir atributos/métodos e construir um Diagrama de Classes que represente a estrutura lógica do sistema.
* **Padrões de Projeto:** Analisar a aplicação (ou potencial aplicação) de padrões como *MVC*, *Factory* e *Singleton*.
* **Crítica Técnica:** Identificar limitações e propor melhorias arquiteturais para aumentar a escalabilidade e manutenção do software.
