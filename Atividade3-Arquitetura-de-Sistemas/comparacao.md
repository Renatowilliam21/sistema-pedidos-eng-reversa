Parte 5 – COMPARAÇÃO DO SISTEMA DIDÁTICO

Arquitetura do sistema real:
Arquitetura web cliente-servidor, utilizando HTML, CSS, JavaScript.

Arquitetura do sistema didático:
Arquitetura simples apenas no lado do cliente (JavaScript), sem servidor ou banco de dados.

Coesão do sistema real:
A interface demonstra boa organização das funcionalidades, com áreas específicas para cardápio, categorias de produtos e carrinho.

Coesão do sistema didático:
As funcionalidades estão concentradas em poucas operações, focadas apenas em adicionar itens, calcular valores e finalizar pedidos.

Acoplamento do sistema real:
Pela interface, as funcionalidades parecem relativamente independentes, como navegação entre categorias e visualização do carrinho.

Acoplamento do sistema didático:
As operações parecem depender diretamente umas das outras, já que o cálculo de total, desconto e finalização fazem parte de um mesmo fluxo simples.

Organização do sistema real:
O sistema apresenta uma organização clara do cardápio, separando produtos em categorias como pizzas, sanduíches, bebidas e porções.

Organização do sistema didático:
A organização é mais simples, baseada apenas em uma lista de itens adicionados e no cálculo do total do pedido.

Flexibilidade do sistema real:
O sistema parece mais flexível, permitindo adicionar novas categorias de produtos, promoções ou combos.

Flexibilidade do sistema didático:
Menos flexível, pois os produtos e valores parecem definidos previamente e com poucas opções de expansão.
