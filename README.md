- Arquitetura de Software

- Versionamento com Git

## Desafio

Você deverá:

1. Analisar o sistema existente  
2. Identificar problemas estruturais  
3. Propor melhorias  
4. Refatorar o código  
5. Aplicar padrões de projeto  
6. Utilizar Git corretamente durante o processo

## O que analisar

Durante a análise, identifique problemas relacionados a:

- Baixa coesão  
- Alto acoplamento  
- Falta de separação de responsabilidades  
- Duplicação de código  
- Organização do sistema

## Implementações obrigatórias

Você deve aplicar no sistema:

### Factory
Utilizar para criação de objetos (ex: produtos, pedidos)

### Singleton
Utilizar para garantir uma única instância (ex: controle do pedido ou persistência)

## **Descrição da Atividade**

Este projeto tem como propósito a análise de um sistema real de pedidos online, aplicando conceitos fundamentais de engenharia de software, arquitetura de sistemas e padrões de projeto.

A atividade consiste em identificar a estrutura do sistema, compreender sua arquitetura, avaliar decisões de design, realizar a modelagem de classes e propor melhorias. Todo o processo simula um cenário prático de engenharia reversa, amplamente utilizado no contexto profissional.

---

## **Objetivo do Projeto**

O objetivo principal é compreender o funcionamento interno de um sistema de pedidos online, analisando:

* Suas principais entidades  
* Organização arquitetural  
* Nível de acoplamento e coesão  
* Padrões de projeto aplicados ou inferidos

Além disso, busca-se propor melhorias estruturais com base em boas práticas da engenharia de software.

---

## **Estrutura do Projeto**

O repositório está organizado da seguinte forma:

* **analise.md**  
   Contém a análise detalhada do sistema real, incluindo objetivos, funcionalidades, arquitetura, design e padrões de projeto.  
* **comparacao.md**  
   Apresenta uma comparação entre o sistema real e um modelo didático baseado em boas práticas.  
* **modelagem.png / PDF**  
   Diagrama UML com classes, atributos, métodos e relacionamentos do sistema.  
* **arquitetura.md**  
   Proposta de melhoria arquitetural, com foco na organização em camadas e na separação de responsabilidades.  
* **melhorias.md**  
   Identificação de problemas e reflexões críticas sobre possíveis melhorias.
